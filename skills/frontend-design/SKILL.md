# Frontend Design

Create production-quality interface work.

1. Start from the existing design system, component library, routing, state, and styling conventions.
2. Design for the real user workflow before decoration. Make common actions obvious, efficient, and responsive.
3. Use established controls for common interactions: icons for tools, toggles for binary settings, menus for option sets, tabs for views, and sliders or inputs for numeric values.
4. Ensure text fits its containers on mobile and desktop. Avoid overlapping UI, layout shifts, one-note palettes, and decorative clutter.
5. Verify important visual states with available tests or screenshots when the change affects layout, responsiveness, animation, or interaction.

Keep the first screen useful. Do not replace an app or tool request with a marketing landing page unless the user explicitly asks for one.
